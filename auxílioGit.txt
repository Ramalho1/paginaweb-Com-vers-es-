é necessário lembrar, que quando se usa na primeira vez o git num computador, deve-se configurar as variáveis globais. Vejamos:

	PRIMEIRO - DEVE-SE NO vs.CODE modificar de powershell para bash - Então damos o git init (leia abaixo antes)
		***TAMBÉM, É INTERESSANTE DAR UM git --version pra notar antes se de fato existe o git instalado



				Etapa 1 — Configurar seu nome e e-mail do Git

Abra o Git Bash e digite esses dois comandos (substitua pelos seus dados do GitHub):

git config --global user.name "André Ramalho"
git config --global user.email "seuemail@exemplo.com"


<<<Importante>>>
Coloque o mesmo e-mail que você usa no GitHub (aquele que aparece no seu perfil).
Exemplo:

git config --global user.email "andreramalho@gmail.com"


				Etapa 2 — Confirme se deu certo

Digite:

git config --global --list


Você deve ver algo como:

user.name=André Ramalho
user.email=andreramalho@gmail.com

				Etapa 3 — Refaça o commit

Agora volte para sua pasta do projeto (primeiro) e repita:

git add .
git commit -m "primeira versão"


Pronto!
Depois disso, você poderá seguir normalmente com:


git remote add origin https://github.com/Ramalho1/paginaweb.git   (CONECTA A PASTA AO REPOSITÓRIO)
						"esse endereço deve ser o mesmo que aparece no repositório criado"

		ANTES DESSA PARTE ABAIXO, PODE SER QUE ELE SOLICITE ENTRAR NA SUA CONTA. CLIQUE PRA ENTRA "WITH" O BROWSER


git branch -M main  (ENVIAR OS ARQUIVOS... QUE É FAZER O PUSH) Só se faz uma vez

git push -u origin main  (Somente aqui os arquivos são enviados)
Sempre usaremos essa ultima etapa...
A sequência é:
git add .
git commit -m
git push
